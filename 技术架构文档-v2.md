## 1. 整体架构概览

```
┌───────────────────────────────────────────────────────────────────┐
│                         前端（React SPA）                          │
│  AgentDetailPage → TrainingRunModal → ProposalReviewModal          │
│  CaseManagement → RubricBuilder → TraceViewer                      │
└──────────────────────────┬────────────────────────────────────────┘
                           │ REST / SSE
┌──────────────────────────▼────────────────────────────────────────┐
│                     API Gateway（FastAPI）                          │
└─┬────────┬──────────┬──────────┬──────────┬──────────┬────────────┘
  │        │          │          │          │          │
  ▼        ▼          ▼          ▼          ▼          ▼
Agent   Training   Proposal   Eval/Judge  Version  Retrieval
Service  Engine    Service    Service     Store    Service
  │        │                    │
  │   ┌────┴──────────────┐     │
  │   │  ClusterService   │     │
  │   │  ProposerService  │     │
  │   │  CriticService    │     │
  │   └───────────────────┘     │
  │                             │
  │   ┌─────────────────────┐   │
  │   │  RubricGenService   │   │
  │   │  CaseGenService     │   │
  │   │  JudgeCalibService  │   │
  │   └─────────────────────┘   │
  │                             │
  └─────────────┬───────────────┘
                ▼
         PostgreSQL + pgvector
         Redis（Pub/Sub + 缓存 + 任务队列）
         S3（版本快照 + Trace 归档）
         Langfuse（Trace 存储与展示）
```

**架构关键原则：**

1. **无状态计算层**：所有 Service 无状态，通过 Redis 任务队列调度，水平扩展。
2. **不可变版本**：所有配置变更（训练 + 手动编辑）均产生不可变 AgentVersion 快照。
3. **规则隔离**：训练产生的规则提案不直接写入共享规则库，先创建为 Agent 私有草稿。
4. **分级 Judge**：Judge 校准分三阶段渐进，不设硬门槛阻断冷启动。
5. **贪心守门**：回归门控失败时，优先按风险分贪心剔除，而非全量 ablation。

### 2.9 工作流步骤数据模型（WorkflowStep / OutputAnchor / ConstraintRule）

工作流步骤作为 `agent_whitebox.workflow` 字段存储（JSONB 数组）。以下 TypeScript 类型描述其结构，同时作为 API 和前端的共享 schema。

**ConstraintRule**（跨字段约束，点选 DSL）：

```typescript
interface ConstraintRule {
  id: string;
  ifField: string;                             // 条件字段 key
  ifOp: "eq" | "neq" | "gt" | "gte" | "lt" | "lte";
  ifValue: string | boolean | number;
  thenField: string;                           // 受约束字段 key
  thenMustBe: string | boolean | number;
}
```

**AnchorField**（输出锚点字段，约束改为结构化 DSL）：

```typescript
interface AnchorField {
  id: string;
  key: string;
  type: "enum" | "bool" | "text" | "number" | "list";
  options?: string[];        // 仅当 type = "enum"
  required: boolean;
  description: string;
  constraintDSL?: ConstraintRule[];  // 取代原自然语言 constraint 字符串
}
```

**RoutingRule**（路由规则，条件改为结构化三元组）：

```typescript
interface RoutingRule {
  id: string;
  conditionDSL?: {           // 取代原自然语言 condition 字符串
    field: string;
    op: "eq" | "neq" | "gt" | "lt";
    value: string | boolean | number;
  };
  nextStepId: string;        // "__end__" 表示终止流程
}
```

**WorkflowStep**（工作流步骤，新增 `inputFields` 声明）：

```typescript
interface WorkflowStep {
  id: string;
  title: string;
  description: string;
  enabled: boolean;
  skillRef?: string;          // 引用 Skill 库中的 Skill ID
  inputFields?: string[];     // 声明本步骤读取哪些上游 anchor field key（按名匹配）
  anchor?: {
    strict: boolean;
    fields: AnchorField[];
  };
  routing?: RoutingRule[];
}
```

**Constraint DSL 评估（执行引擎，O(1)）**：约束在步骤输出锚点填充完成后立即由执行引擎评估，不走 LLM：

```python
OPERATORS = {
    "eq": lambda a, b: a == b, "neq": lambda a, b: a != b,
    "gt": lambda a, b: a > b,  "gte": lambda a, b: a >= b,
    "lt": lambda a, b: a < b,  "lte": lambda a, b: a <= b,
}

def evaluate_constraint(rule: ConstraintRule, output: dict) -> bool:
    """Returns False if constraint is violated."""
    cond_met = OPERATORS[rule.ifOp](output.get(rule.ifField), rule.ifValue)
    if not cond_met:
        return True   # condition not triggered, constraint not applicable
    return output.get(rule.thenField) == rule.thenMustBe

def evaluate_all_constraints(step: WorkflowStep, output: dict) -> list[ConstraintViolation]:
    violations = []
    if not step.anchor:
        return violations
    for field in step.anchor.fields:
        for rule in (field.constraintDSL or []):
            if not evaluate_constraint(rule, output):
                violations.append(ConstraintViolation(
                    rule_id=rule.id, step_id=step.id,
                    field=rule.thenField, expected=rule.thenMustBe,
                    actual=output.get(rule.thenField)
                ))
    return violations
```

违反约束时，trace 中记录 `constraint_violations: [...]`，可精确定位到步骤、字段和规则 ID，无需事后猜测。

## 3. 训练引擎（七步 + 摘要流水线）

训练引擎协调以下步骤，前 4 步全自动，步骤 5 等待用户输入，步骤 6-7 自动，版本写入后异步生成摘要卡：

```
① 数据准备 → ② 基线评估 → ③ 失败聚类 → ④ Proposer 生成
         → ⑤ 人工审查（阻塞）→ ⑥ 回归守门 → ⑦ 版本写入 → ⑦' 摘要生成
```

### 3.1 任务调度

```python
@celery_app.task(bind=True, max_retries=2)
def run_training_task(self, run_id: str):
    engine = TrainingEngine(run_id)
    try:
        # 步骤 1-4 连续执行
        dataset = engine.prepare_data()
        baseline = engine.evaluate_baseline(dataset)
        clusters = engine.cluster_failures(baseline.failed_cases)
        proposals = engine.generate_proposals(clusters)
        
        # 步骤 5：暂停，等待人工审查
        engine.set_status('awaiting_review')
        engine.publish_event('proposals_ready', {
            'proposal_count': len(proposals),
            'cluster_count': len(clusters)
        })
        # 后续由 submit_review API 触发步骤 6-7
        
    except Exception as exc:
        engine.set_status('failed', error=str(exc))
        raise self.retry(exc=exc, countdown=30)
```

### 3.2 步骤①：数据准备

```python
class DataPreparer:
    MIN_TRAIN = 5
    MIN_HOLDOUT = 3
    
    def prepare(self, agent_id: str) -> PreparedDataset:
        cases = db.get_eval_cases(agent_id)
        
        # 前置检查
        if len(cases) < self.MIN_TRAIN + self.MIN_HOLDOUT:
            raise TrainingError(
                code='INSUFFICIENT_CASES',
                message=f"需要至少 {self.MIN_TRAIN + self.MIN_HOLDOUT} 条案例，"
                        f"当前只有 {len(cases)} 条"
            )
        
        train = [c for c in cases if c.split == 'train']
        holdout = [c for c in cases if c.split == 'holdout']
        
        # 若未划分，自动分层抽样并写回（一次性，之后固定）
        if not train:
            train, holdout = self._stratified_split(cases, holdout_ratio=0.2)
            db.bulk_update_split(train, holdout)
        
        # 记录样本量用于后续可靠性提示
        db.update_training_run_field(self.run_id, holdout_sample_size=len(holdout))
        
        return PreparedDataset(train=train, holdout=holdout)
    
    def _stratified_split(self, cases, holdout_ratio):
        # 按 expected_verdict 分层，确保 holdout 各类别有代表
        pass_cases = [c for c in cases if c.expected_verdict == '通过']
        fail_cases = [c for c in cases if c.expected_verdict != '通过']
        
        holdout = (
            random.sample(pass_cases, max(1, int(len(pass_cases) * holdout_ratio))) +
            random.sample(fail_cases, max(1, int(len(fail_cases) * holdout_ratio)))
        )
        train = [c for c in cases if c not in holdout]
        return train, holdout
```

### 3.3 步骤②：基线评估

```python
async def evaluate_baseline(self, dataset: PreparedDataset) -> BaselineResult:
    agent_config = db.get_agent_whitebox(self.agent_id)
    
    # 并发执行 train + holdout（使用 asyncio.gather）
    train_outputs, holdout_outputs = await asyncio.gather(
        self._run_batch(agent_config, dataset.train),
        self._run_batch(agent_config, dataset.holdout)
    )
    
    train_scores = await judge.score_batch(train_outputs, agent_config.rubric)
    holdout_scores = await judge.score_batch(holdout_outputs, agent_config.rubric)
    
    failed_cases = [o for o in train_outputs if not o.passed]
    
    # 附带 Rubric 哈希，确保后续评估可比性
    return BaselineResult(
        train_score=train_scores.weighted_avg,
        holdout_score=holdout_scores.weighted_avg,
        failed_cases=failed_cases,
        rubric_hash=agent_config.rubric_version
    )
```

## 4. 失败聚类服务

失败聚类是 Proposer 的前置步骤，把原始失败案例转化为有诊断意义的失败主题。

### 4.1 聚类算法

```python
class FailureClusterService:
    
    async def cluster(
        self, 
        failed_cases: list[CaseOutput],
        agent_config: AgentWhitebox
    ) -> list[FailureCluster]:
        
        if len(failed_cases) == 0:
            return []
        
        if len(failed_cases) <= 3:
            # 案例太少，不聚类，每条案例单独成一个"簇"
            return [self._single_case_cluster(c) for c in failed_cases]
        
        # 1. 向量化：嵌入 judge_feedback + actual_output 的组合文本
        texts = [f"{c.judge_feedback}\n{c.actual_output}" for c in failed_cases]
        embeddings = await embed_batch(texts)
        
        # 2. 动态 k：k = min(失败数 // 3, 5)，至少 1
        k = max(1, min(len(failed_cases) // 3, 5))
        
        # 3. k-means 聚类
        from sklearn.cluster import KMeans
        km = KMeans(n_clusters=k, random_state=42, n_init=10)
        labels = km.fit_predict(embeddings)
        
        # 4. 对每个簇做 Critic 诊断
        clusters = []
        for cluster_id in range(k):
            cluster_cases = [c for c, l in zip(failed_cases, labels) if l == cluster_id]
            diagnosis = await self._diagnose_cluster(cluster_cases, agent_config)
            
            clusters.append(FailureCluster(
                label=diagnosis.label,
                root_cause=diagnosis.root_cause,
                target_units=diagnosis.target_units,
                case_ids=[c.case_id for c in cluster_cases],
                representative_case_id=self._pick_representative(cluster_cases, km.cluster_centers_[cluster_id], embeddings)
            ))
        
        return clusters
    
    async def _diagnose_cluster(
        self, 
        cases: list[CaseOutput], 
        agent_config: AgentWhitebox
    ) -> ClusterDiagnosis:
        """让 Critic 模型对一个失败簇给出自然语言诊断"""
        
        response = await llm.complete(
            model="claude-sonnet-4-6",  # 诊断用中等模型
            prompt=CLUSTER_DIAGNOSIS_PROMPT.format(
                cases_json=json.dumps([c.to_dict() for c in cases[:5]]),  # 最多 5 条代表案例
                whitebox_summary=agent_config.to_summary(),
                rubric_json=json.dumps(agent_config.rubric)
            )
        )
        return ClusterDiagnosis(**json.loads(response))
    
    def _pick_representative(self, cases, centroid, all_embeddings):
        """选择距离聚类中心最近的案例作为代表案例"""
        case_embeddings = [all_embeddings[cases.index(c)] for c in cases]
        distances = [np.linalg.norm(e - centroid) for e in case_embeddings]
        return cases[np.argmin(distances)].case_id
```

```python
            if all(scores.get(k, 0) >= threshold 
                   for k, threshold in self.CHECKS.items()):
                p.risk_flag = p.risk_score > 0.6
```

```python
async def _analyze_dependencies(self, proposals: list[Proposal]) -> list[Proposal]:
    """识别提案间的互补和冲突关系"""
    
    if len(proposals) <= 1:
        return proposals
    
    response = await llm.complete(
        model="claude-haiku-4-5-20251001",
        prompt=DEPENDENCY_ANALYSIS_PROMPT.format(
            proposals_json=json.dumps([p.to_summary() for p in proposals])
        )
    )
    
    deps = json.loads(response)
    
    # 更新每个提案的 depends_on 和 conflicts_with 字段
    dep_map = {p.id: p for p in proposals}
    for rel in deps.get('complementary', []):
        dep_map[rel['a']].depends_on.append(rel['b'])
        dep_map[rel['b']].depends_on.append(rel['a'])
    for rel in deps.get('conflicts', []):
        dep_map[rel['a']].conflicts_with.append(rel['b'])
        dep_map[rel['b']].conflicts_with.append(rel['a'])
    
    return proposals
```

### 6.2 回归门控（动态阈值 + 贪心 Ablation）

```python
class RegressionGate:
    
    # 动态阈值：基于 holdout 样本量调整
    THRESHOLDS = {
        'small':    {'min_holdout': 3,  'train_min': 0.00, 'holdout_max': -0.05},  # < 15 条
        'medium':   {'min_holdout': 15, 'train_min': 0.01, 'holdout_max': -0.02},  # 15-30 条
        'sufficient': {'min_holdout': 30, 'train_min': 0.02, 'holdout_max': -0.01}, # 30+ 条
    }
    
    def _get_threshold(self, holdout_size: int) -> dict:
        if holdout_size >= 30:
            return self.THRESHOLDS['sufficient']
        elif holdout_size >= 15:
            return self.THRESHOLDS['medium']
        else:
            return self.THRESHOLDS['small']
    
    async def evaluate(
        self, 
        run_id: str, 
        accepted_proposals: list[Proposal],
        baseline: BaselineResult
    ) -> GateResult:
        
        holdout_size = db.get_holdout_size(run_id)
        threshold = self._get_threshold(holdout_size)
        
        # 构建候选配置并评估
        whitebox = db.get_agent_whitebox(self.agent_id)
        candidate = apply_proposals(whitebox, accepted_proposals)
        
        dataset = db.get_dataset(self.agent_id)
        train_score, holdout_score = await asyncio.gather(
            self._score_set(candidate, dataset.train),
            self._score_set(candidate, dataset.holdout)
        )
        
        train_delta = train_score - baseline.train_score
        holdout_delta = holdout_score - baseline.holdout_score
        
        train_ok = train_delta >= threshold['train_min']
        holdout_ok = holdout_delta >= threshold['holdout_max']
        
        if train_ok and holdout_ok:
            return GateResult(
                passed=True,
                train_delta=train_delta,
                holdout_delta=holdout_delta,
                reliability_note=self._reliability_note(holdout_size)
            )
        
        # 门控失败：贪心 ablation
        auto_rejected = await self._greedy_ablation(
            whitebox, accepted_proposals, baseline, dataset, threshold
        )
        
        return GateResult(
            passed=bool(auto_rejected is not None),  # 贪心找到解则局部通过
            train_delta=train_delta,
            holdout_delta=holdout_delta,
            auto_rejected_ids=[p.id for p in auto_rejected] if auto_rejected else [],
            reliability_note=self._reliability_note(holdout_size)
        )
    
    async def _greedy_ablation(
        self, whitebox, proposals, baseline, dataset, threshold
    ) -> list[Proposal] | None:
        """
        按 risk_score 降序贪心剔除提案，直到找到通过门控的最小剔除集合。
        最坏情况 O(N)，但通常 1-2 次即可找到。
        """
        # 按风险从高到低排序
        sorted_proposals = sorted(proposals, key=lambda p: p.risk_score, reverse=True)
        rejected_so_far = []
        
        for p in sorted_proposals:
            rejected_so_far.append(p)
            remaining = [x for x in proposals if x not in rejected_so_far]
            
            if not remaining:
                return None  # 全部剔除也无法通过，放弃
            
            candidate = apply_proposals(whitebox, remaining)
            _, holdout_score = await asyncio.gather(
                self._score_set(candidate, dataset.train),
                self._score_set(candidate, dataset.holdout)
            )
            
            holdout_delta = holdout_score - baseline.holdout_score
            if holdout_delta >= threshold['holdout_max']:
                return rejected_so_far  # 找到最小剔除集合
        
        return None  # 所有提案都剔除了也不行
    
    def _reliability_note(self, holdout_size: int) -> str:
        if holdout_size < 5:
            return "⚠️ 保留集样本极少（< 5 条），门控结论仅供参考"
        elif holdout_size < 15:
            return "⚡ 保留集样本较少，建议积累到 15+ 条后训练结论更可信"
        elif holdout_size < 30:
            return "✅ 保留集样本良好，结论有参考价值"
        else:
            return "✅ 保留集样本充分，结论统计可信"
```

```python
class JudgePhaseManager:
    """管理每个 Agent 的 Judge 校准阶段"""
    
    PHASE_THRESHOLDS = {
        'human':    {'max_human_judgments': 20},   # 纯人工：前 20 条
        'parallel': {'min_human': 20, 'max_human': 50},  # 人工 + LLM 并行
        'auto':     {'min_human': 50}              # LLM 主导
    }
    
    def get_current_phase(self, agent_id: str) -> str:
        count = db.get_human_judgment_count(agent_id)
        if count < 20:
            return 'human'
        elif count < 50:
            return 'parallel'
        else:
            return 'auto'
    
    def should_advance_phase(self, agent_id: str) -> tuple[bool, str]:
        """检查是否应该升级到下一阶段"""
        current = db.get_agent_judge_phase(agent_id)
        
        if current == 'human':
            count = db.get_human_judgment_count(agent_id)
            if count >= 20:
                return True, 'parallel'
        
        elif current == 'parallel':
            count = db.get_human_judgment_count(agent_id)
            kappa = self.compute_kappa(agent_id)
            # 需要人工判断 ≥ 50 条 且 kappa ≥ 0.6
            if count >= 50 and (kappa is None or kappa >= 0.6):
                return True, 'auto'
        
        return False, current
```

### 7.2 Judge 评分流程

```python
class JudgeService:
    
    async def score_case(
        self, 
        agent_id: str,
        case: EvalCase, 
        output: str,
        rubric: list[RubricCriterion]
    ) -> JudgeScore:
        
        phase = db.get_agent_judge_phase(agent_id)
        
        if phase == 'human':
            # 纯人工阶段：不调用 LLM Judge，等待人工判断
            return JudgeScore(source='pending_human', score=None)
        
        # parallel 或 auto 阶段：调用 LLM Judge
        response = await llm.complete(
            model="claude-sonnet-4-6",
            prompt=JUDGE_PROMPT.format(
                rubric=rubric,
                input=case.input,
                expected=case.expected_reason,
                output=output
            )
        )
        
        score = JudgeScore(**json.loads(response), source='llm')
        
        # parallel 阶段：同时创建校准任务（抽样 8%）
        if phase == 'parallel':
            if random.random() < 0.08:
                db.create_calibration_task(
                    agent_id=agent_id,
                    case_judgment_id=score.judgment_id
                )
        
        return score
    
    def compute_kappa(self, agent_id: str) -> float | None:
        """计算最近 100 条对齐的 LLM vs 人工判断的 Cohen's Kappa"""
        from sklearn.metrics import cohen_kappa_score
        
        completed = db.get_completed_calibration_tasks(agent_id, limit=100)
        if len(completed) < 10:
            return None
        
        llm_labels = [t.llm_judgment for t in completed]
        human_labels = [t.human_judgment for t in completed]
        return cohen_kappa_score(llm_labels, human_labels)
    
    def get_calibration_status(self, agent_id: str) -> CalibrationStatus:
        kappa = self.compute_kappa(agent_id)
        phase = db.get_agent_judge_phase(agent_id)
        
        if kappa is None:
            return CalibrationStatus(
                phase=phase, kappa=None,
                message="累积更多判断后将自动计算一致率"
            )
        
        healthy = kappa >= 0.6
        return CalibrationStatus(
            phase=phase, kappa=kappa,
            healthy=healthy,
            message="" if healthy else 
                    f"LLM 与人工判断一致率偏低（κ={kappa:.2f}），"
                    f"建议检查 Rubric 定义是否清晰"
        )
```

### 10.2 UUID 驱动的 Few-shot Diff

```python
def diff_versions(v1: AgentVersion, v2: AgentVersion) -> VersionDiff:
    s1 = v1.snapshot['whitebox']
    s2 = v2.snapshot['whitebox']
    
    diffs = []
    
    # ✅ Few-shot diff：UUID 匹配（不再使用 input[:50]）
    fs1 = {fs['id']: fs for fs in s1.get('few_shots', [])}
    fs2 = {fs['id']: fs for fs in s2.get('few_shots', [])}
    
    all_ids = set(fs1) | set(fs2)
    for fid in all_ids:
        if fid not in fs1:
            diffs.append(DiffEntry(unit='few-shot', type='added', 
                                   after=fs2[fid], before=None))
        elif fid not in fs2:
            diffs.append(DiffEntry(unit='few-shot', type='removed', 
                                   before=fs1[fid], after=None))
        elif fs1[fid] != fs2[fid]:
            # 捕获 output 变化（input 不变但 output 改了）
            diffs.append(DiffEntry(unit='few-shot', type='modified',
                                   before=fs1[fid], after=fs2[fid]))
    
    # Instruction Segments diff（同样用 UUID 匹配）
    segs1 = {s['id']: s for s in s1.get('instruction_segs', [])}
    segs2 = {s['id']: s for s in s2.get('instruction_segs', [])}
    for sid in set(segs1) | set(segs2):
        if segs1.get(sid) != segs2.get(sid):
            diffs.append(DiffEntry(
                unit='instruction', type='modified',
                before=segs1.get(sid), after=segs2.get(sid)
            ))
    
    # Retrieval config diff
    rc1 = s1.get('retrieval_config', {})
    rc2 = s2.get('retrieval_config', {})
    if rc1 != rc2:
        diffs.append(DiffEntry(unit='retrieval', type='modified',
                               before=rc1, after=rc2))
    
    # Rubric diff
    rub1 = {r['id']: r for r in s1.get('rubric', [])}
    rub2 = {r['id']: r for r in s2.get('rubric', [])}
    if rub1 != rub2:
        diffs.append(DiffEntry(unit='rubric', type='modified',
                               before=list(rub1.values()), after=list(rub2.values())))
    
    # 附带 Rubric 一致性检查
    rubric_changed = v1.rubric_hash != v2.rubric_hash
    
    return VersionDiff(
        from_version=v1.version_tag,
        to_version=v2.version_tag,
        entries=diffs,
        rubric_changed=rubric_changed,
        rubric_comparability_warning=(
            "⚠️ Rubric 在此期间有修改，两版本的评估分数使用了不同标准，直接对比可能有误导"
            if rubric_changed else None
        )
    )
```

```python
                    "修改共享规则需要资产维护者权限，已转为变更申请"
```

```python
class AgentRetriever:
    
    async def retrieve(
        self, 
        query: str, 
        agent_id: str,
        config: RetrievalConfig
    ) -> list[RetrievedChunk]:
        
        query_embedding = await embed(query)
        
        # 参数化查询（防止 SQL 注入）
        results = await db.query(
            """
            SELECT doc_id, chunk_index, content,
                   1 - (embedding <=> $1::vector) AS similarity
            FROM knowledge_embeddings
            WHERE $2 = ANY(linked_agent_ids)
              AND ($3::text[] IS NULL OR tags && $3::text[])
            ORDER BY embedding <=> $1::vector
            LIMIT $4
            """,
            query_embedding, agent_id, 
            config.tag_filters or None,
            config.top_k
        )
        
        return [RetrievedChunk(
            doc_id=r.doc_id,
            content=r.content,
            similarity=r.similarity
        ) for r in results]
    
    async def test_retrieval(
        self, query: str, agent_id: str
    ) -> list[RetrievedChunk]:
        """检索测试面板使用，返回更多结果和调试信息"""
        config = db.get_retrieval_config(agent_id)
        chunks = await self.retrieve(query, agent_id, config)
        
        # 同时检查"未命中但相关"的文档（忽略 tagFilters 重试）
        if config.tag_filters:
            without_filter = await self.retrieve(
                query, agent_id,
                RetrievalConfig(top_k=config.top_k, tag_filters=[])
            )
            missed = [c for c in without_filter if c not in chunks]
        else:
            missed = []
        
        return RetrievalTestResult(hits=chunks, missed_by_filter=missed)
```

### 14.1 Trace 架构

使用 Langfuse 作为 LLM 调用的 Trace 后端，所有关键步骤用 decorator 标注：

```python
from langfuse.decorators import observe, langfuse_context

@observe(name="agent_inference")
async def run_agent_on_case(
    agent_config: AgentWhitebox, 
    case: EvalCase,
    run_id: str
) -> CaseOutput:
    
    # 绑定 test_case_id，使 Trace 可以按案例查询
    langfuse_context.update_current_trace(
        metadata={
            "test_case_id": str(case.id),
            "agent_id": str(agent_config.agent_id),
            "training_run_id": run_id
        }
    )
    
    # Step 1: 知识检索
    with langfuse_context.current_observation() as span:
        span.name = "knowledge_retrieval"
        retrieved = await retriever.retrieve(
            case.input, agent_config.agent_id, agent_config.retrieval_config
        )
        span.output = {
            "chunks_count": len(retrieved),
            "top_similarity": retrieved[0].similarity if retrieved else 0,
            "docs": [{"doc_id": c.doc_id, "similarity": c.similarity} for c in retrieved]
        }
    
    # Step 2: 规则校验
    with langfuse_context.current_observation() as span:
        span.name = "rule_evaluation"
        rule_results = await evaluate_rules(case.input, agent_config.rules)
        span.output = {
            "applied_rules": [r.rule_id for r in rule_results if r.triggered],
            "skipped_rules": [r.rule_id for r in rule_results if not r.triggered]
        }
    
    # Step 3: LLM 推理
    with langfuse_context.current_observation() as span:
        span.name = "llm_reasoning"
        prompt = build_prompt(agent_config, case.input, retrieved, rule_results)
        response = await llm.complete(model=agent_config.model, prompt=prompt)
        span.input = {"prompt_length": len(prompt)}
        span.output = {"response_length": len(response)}
        span.usage = response.usage
    
    return CaseOutput(
        case_id=case.id,
        output=response.content,
        trace_id=langfuse_context.get_current_trace_id()
    )
```

```python
async def stream_training_events(run_id: str, request: Request):

        pubsub = redis.pubsub()
        pubsub.subscribe(f"training:{run_id}:events")
        
        # 先发送当前状态（防止订阅前已有事件）

                if data.get('status') in ('done', 'failed', 'awaiting_review'):
                    break
        
        pubsub.unsubscribe(f"training:{run_id}:events")
    
    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"}
```

```python
# 训练成本估算（在启动训练前展示给用户）
def estimate_training_cost(agent_id: str) -> CostEstimate:
    n_train = db.get_train_case_count(agent_id)
    n_holdout = db.get_holdout_case_count(agent_id)
    
    # 各步骤 token 估算（基于案例数量）
    baseline_calls = n_train + n_holdout  # 基线评估
    proposer_calls = 1                    # Proposer（一次调用处理所有聚类）
    gate_calls = n_holdout                # 回归守门
    worst_case_ablation = 5 * n_holdout   # 最坏情况 Ablation（估算 5 条提案）
    
    total_calls = baseline_calls + proposer_calls + gate_calls + worst_case_ablation
    
    # 按各步骤模型计费
    estimated_usd = (
        baseline_calls * 0.003 +    # Sonnet
        proposer_calls * 0.015 +    # Opus
        gate_calls * 0.003 +        # Sonnet
        worst_case_ablation * 0.003 # Sonnet
    )
    
    return CostEstimate(
        calls=total_calls,
        estimated_usd=estimated_usd,
        worst_case_usd=estimated_usd * 1.5  # 留 50% 余量
    )
```

## 19. 附录：v1.0 → v2.0 变更映射

| 问题 | v1.0 | v2.0 修订 |
|------|------|----------|
| 小数据集门控失效 | 固定阈值 2% / 1% | 动态阈值按 holdout 样本量，附可靠性提示 |
| Judge 校准硬门槛 | 一致率 < 0.6 阻断训练 | 三阶段渐进策略（human → parallel → auto）|
| Ablation O(N) 成本 | 全量 leave-one-out | 贪心 ablation（按 risk_score 排序）|
| 规则跨 Agent 污染 | 无隔离 | 三类 rule Proposal + 私有草稿路径 |
| WebSocket 轮询 | 每秒轮询 Redis | Redis Pub/Sub + SSE |
| Few-shot diff 用 input[:50] | 截断匹配，有碰撞 | UUID 驱动匹配 |
| 失败聚类缺失 | Proposer 直接消费原始失败列表 | ClusterService 先聚类，Proposer 消费簇 |
| 冷启动案例生成缺失 | 用户自行创建 | 锚点案例 + 知识库采样 + 覆盖率引导生成 |
| Rubric 生成缺实现 | 无 | RubricGenService（多轮对话，Redis 会话状态）|
| 手动编辑不产生版本 | 只有训练产生版本 | 白盒 PUT 接口自动创建 manual-edit 版本 |
| Step 级 Trace 缺失 | 仅 Structured Logging | Langfuse 集成，trace 与 test_case_id 关联 |
| 提案冲突检测缺失 | 无 | depends_on / conflicts_with 字段 + Proposer 依赖分析 |
| Rubric 历史分数可比性 | 无 | EvalRun 附带 rubric_hash，版本 diff 给出不可比警告 |
| 程序性记忆 v1 过重 | 复杂写门 + 时间衰减 + 语义去重 | v1 简化为单 Agent 数据库记录，v2 升级 |
